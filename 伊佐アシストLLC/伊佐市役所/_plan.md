## 今回のスコープ（フロントエンドデモ）

### スコープ内（制作する）
- タッチスクリーン端末向けUI（横向き・縦向き両対応、想定解像度1024×768以上）
- ホーム画面：カテゴリ選択タイル
- FAQ検索・一覧・詳細表示
- キーワード入力フォーム（テキスト）
- 多言語切り替えボタン（日本語・英語・中国語・韓国語 の4言語、静的テキスト切り替え）
- アイドルタイムアウト（60秒で自動ホーム遷移）
- 「番号札を発行する」「職員を呼ぶ」ボタン（遷移のみ、実際の発行処理はなし）
- GSAPを使ったローディングスピナーアニメーション
- 文字サイズ拡大トグルボタン
- フロアマップ表示画面（静的画像）
- WCAG 2.1 AA準拠のアクセシビリティ対応（コントラスト・タッチターゲット・aria属性）

### スコープ外（今回制作しない）
- バックエンド・データベース（PostgreSQL、pgvector等）
- OpenAI API / LLM連携（RAG構成）
- 管理画面（FAQ CRUD・ダッシュボード・ユーザー管理）
- 番号札発行ハードウェア連携
- 職員呼び出し通知（実際の通知送信）
- 音声入力（Web Speech API）
- JWT認証・IPアドレス制限
- QRコードスキャン機能（QRコード画像の表示のみ）

## デザイナーへの指示

### 全体イメージ
- 公共機関として「信頼感・清潔感・わかりやすさ」を最優先にする
- 高齢者・外国人が戸惑わないシンプルなUI
- 伊佐市のイメージカラー（山・自然・温泉）を参考に、落ち着いた緑系または青緑系をベースカラーとして検討する
- 過度な装飾は避け、余白を十分に取ったレイアウト

### 配色方針
- プライマリカラー: 伊佐市をリサーチして決定（青系・緑系が想定される）
- テキスト: コントラスト比 4.5:1 以上を必ず確保（WCAG 2.1 AA）
- 背景: 白または極薄のグレー（#F8F9FA程度）
- エラー/警告色: 赤系（コントラスト確保必須）

### タイポグラフィ
- 本文フォント最小サイズ: 18px（WCAG要件）
- 見出し: 28px 以上
- フォントファミリー: Noto Sans JP（Google Fonts CDN）+ システムフォント fallback
- 行間: 1.6以上

### タッチ操作対応
- ボタン・タップターゲット最小サイズ: 48×48px（WCAG 2.1 AA）
- ホバー演出は使わない（タッチデバイス専用設計）
- タップ時のフィードバック: CSS active状態で色変化・スケール変化

### カテゴリタイル（ホーム画面）
以下のカテゴリを大きなタイルで表示する:
1. 住民票・戸籍
2. 税金・国保
3. 子育て・教育
4. 介護・福祉
5. 土地・建物
6. ごみ・環境
7. フロアマップ
8. その他・よくある質問

### 成果物
- `index.html`（ホーム画面の完全なHTML+CSS）
- `css/style.css`（全画面共通スタイル + ホーム画面スタイル）
- `_design-research.md`（リサーチ結果・配色根拠・参考URL）

## エンジニアへの指示

### 前提
- デザイナーが作成した `index.html` / `css/style.css` を受け取り、JavaScript機能を実装する
- 既存のHTMLを壊さず、`js/main.js` に機能を追加する

### 実装必須機能

#### 1. 多言語切り替え（静的）
- 対応言語: 日本語（ja）・英語（en）・中国語（zh）・韓国語（ko）
- 実装方式: HTMLの各テキスト要素に `data-i18n="key"` 属性を付与し、JSでテキストを差し替える
- 言語データは `js/i18n.js` に分離して定義する（オブジェクトリテラル形式）
- 選択言語は `localStorage` に保存し、次回アクセス時に維持する

#### 2. アイドルタイムアウト
- 60秒間タッチ・クリック・キー入力がない場合、`index.html` にリダイレクト
- タイムアウト5秒前にカウントダウン表示（オーバーレイ）
- 「キャンセル」ボタンでタイムアウトをリセット可能

#### 3. GSAPローディングアニメーション
- CDN: `https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.2/gsap.min.js`
- ページ遷移時にスピナー表示（最大1秒）
- ホーム画面の各カテゴリタイルは stagger アニメーションで順番にフェードイン

#### 4. FAQ検索（クライアントサイド）
- ダミーFAQデータを `js/faq-data.js` に定義（各カテゴリ5件以上、計40件以上）
- キーワード入力でリアルタイムフィルタリング（300ms debounce）
- カテゴリボタンでフィルタリング
- 検索結果がゼロの場合「エスカレーション画面」へ誘導するメッセージ表示

#### 5. 文字サイズ切り替え
- ホーム画面常設のボタン（「文字を大きく」トグル）
- `document.documentElement` に class `font-large` を付与/除去
- `font-large` 時: 全体フォントサイズを 1.25rem 基準に拡大
- 状態は `localStorage` に保存

#### 6. faq.html / faq-detail.html / floor-map.html / escalate.html
- デザイナーのスタイルを踏襲しつつ、各画面のHTMLを実装する
- `faq-detail.html` は URLパラメータ（`?id=xxx`）でFAQを特定し動的表示

### 技術要件
- `'use strict'` を全JSファイルの先頭に記載
- インデント: スペース2つ
- `const` / `let` のみ使用（`var` 禁止）
- DOM操作はバニラJS（jQueryは使わない）

### 成果物
- `js/main.js`（共通処理: 多言語・タイムアウト・フォントサイズ・GSAPアニメーション）
- `js/i18n.js`（多言語テキスト定義）
- `js/faq-data.js`（ダミーFAQデータ）
- `faq.html`（FAQ検索・一覧画面）
- `faq-detail.html`（FAQ詳細画面）
- `floor-map.html`（フロアマップ画面）
- `escalate.html`（エスカレーション画面）

## ライターへの指示

### 前提
- エンジニアが実装した全HTMLファイルを受け取り、コンテンツ（テキスト）を確認・改善する
- AIデモ用のダミーコンテンツとして実在感のある自然な日本語にする

### 作業内容

#### 1. FAQコンテンツの品質確認・改善（`js/faq-data.js`）
- エンジニアが作成したダミーFAQを確認し、市民目線で自然な質問文・回答文に改善する
- 各カテゴリ最低5件、全体40件以上を維持する
- 回答文は「結論ファースト」で書く（最初の1文で答えを述べる）
- 専門用語は平易な言葉に置き換える（例：「証明書の交付」→「書類をもらう」）
- 高齢者にも理解しやすい文体（読み仮名が必要な漢字は避ける、または括弧で読みを添える）

#### 2. 多言語テキストの確認（`js/i18n.js`）
- 日本語テキストを確認し、自然な表現に修正する
- 英語テキストは簡潔で明快なものになっているか確認する
- （中国語・韓国語はエンジニアが機械翻訳で作成したものをそのまま使用可）

#### 3. UI上のコピーライティング
- ホーム画面のウェルカムメッセージ（例：「何かお困りですか？」）
- エスカレーション画面のメッセージ（番号札発行・職員呼び出しへの誘導文）
- タイムアウト警告メッセージ
- 検索結果ゼロ時のメッセージ

#### トーン・文体
- 敬体（です・ます調）で統一する
- 堅すぎず、親しみやすいが丁寧なトーン
- 一文は50文字以内を目安にする
- 否定形より肯定形を使う（例：「わかりません」→「職員にお尋ねください」）

### 成果物
- 改善済みの `js/faq-data.js`
- 改善済みの `js/i18n.js`
- 各HTMLファイルのUIコピー修正（直接HTMLを編集する）

## テスターへの指示

### 確認観点

#### 1. アクセシビリティ（最重要）
- [ ] 全インタラクティブ要素のタッチターゲットが 48×48px 以上か
- [ ] テキストのコントラスト比が 4.5:1 以上か（Chrome DevTools / Axeで確認）
- [ ] フォントサイズが最小 18px か
- [ ] 主要要素に `aria-label` / `role` 属性が付与されているか
- [ ] `lang` 属性が適切に設定されているか（言語切り替え時も `<html lang="">` が更新されるか）

#### 2. 機能動作確認
- [ ] 全5画面間の遷移が正常に動作するか
- [ ] 多言語切り替えで全テキストが切り替わるか（日本語/英語/中国語/韓国語）
- [ ] 言語設定が `localStorage` に保存され、ページリロード後も維持されるか
- [ ] アイドルタイムアウト（60秒）が動作するか
- [ ] タイムアウト5秒前のカウントダウンが表示されるか
- [ ] カウントダウン中の「キャンセル」ボタンでタイマーがリセットされるか
- [ ] FAQ検索（キーワード入力）でフィルタリングが動作するか
- [ ] カテゴリボタンでのフィルタリングが動作するか
- [ ] 検索結果ゼロ時のメッセージが表示されるか
- [ ] 文字サイズ切り替えが動作し、localStorage に保存されるか
- [ ] GSAPアニメーション（スピナー・stagger）が動作するか

#### 3. レスポンシブ・表示確認
- [ ] 横向き（landscape）1024×768 で崩れないか
- [ ] 縦向き（portrait）768×1024 で崩れないか
- [ ] タブレット想定（iPad相当）のブラウザ幅で確認する

#### 4. コード品質
- [ ] HTMLのバリデーション（W3C Validator相当のチェック）
- [ ] JavaScriptのエラーがブラウザコンソールに出ていないか
- [ ] 使われていないCSSルールがないか（軽微な場合は許容）
- [ ] `var` が使われていないか
- [ ] インデントがスペース2つに統一されているか

#### 5. ブラウザ確認
- Chrome 最新版（タブレット向けなのでChromiumベースを基準）
- Safari（iPadOS）での表示確認も推奨

### 差し戻し基準
以下の場合は該当エージェントへ差し戻す:
- アクセシビリティ要件（タッチターゲット・コントラスト・フォントサイズ）を1件でも満たしていない場合
- JSエラーがコンソールに発生している場合
- 画面遷移が1つでも機能していない場合
- 多言語切り替えが動作しない場合
