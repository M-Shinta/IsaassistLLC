# 伊佐アシストLLC — Web制作ワークフロー

Webサイト制作を専門とする5つのエージェントが協力して、クライアント案件を進めます。

---

## ディレクトリ構成

```
伊佐アシストLLC/
├── README.md                  # このファイル
├── CLAUDE.md                  # Claude へのルール定義
├── .claude/
│   └── agents/                # 専門エージェント定義
│       ├── manager.md
│       ├── designer.md
│       ├── engineer.md
│       ├── writer.md
│       └── tester.md
└── {クライアント名}/            # 案件ごとのフォルダ
    ├── index.html
    ├── css/
    ├── js/
    └── images/
```

---

## 5つのエージェント

| エージェント | 役割 | 主な担当 |
|------------|------|---------|
| **マネージャー** | プロジェクト統括 | 要件整理・構成計画・進行管理 |
| **デザイナー** | UI/UX設計 | HTML構造・CSS・レイアウト・配色 |
| **エンジニア** | 実装・技術 | JavaScript・パフォーマンス・SEO技術面 |
| **ライター** | コンテンツ制作 | 日本語コピー・見出し・メタ情報 |
| **テスター・批評家** | 品質管理 | 検証・レビュー・改善提案 |

---

## 新規案件の進め方

1. **マネージャー**にクライアント情報と要件を伝える
2. マネージャーが構成計画を立て、各エージェントにタスクを割り振る
3. **デザイナー**がHTML/CSSを構築
4. **エンジニア**がJavaScript・インタラクションを実装
5. **ライター**がコンテンツを作成
6. **テスター・批評家**が全体をレビューし、改善点を報告
7. マネージャーが最終確認・納品

---

## クライアントフォルダの命名規則

- 形式: `{クライアント名}` （例: `松江管材株式会社`）
- スペースなし、正式名称を使用
- 1案件 = 1フォルダ
