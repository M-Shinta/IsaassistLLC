# 伊佐アシストLLC — Claude ルール定義

## プロジェクト概要

Web制作会社として、クライアントのWebサイトを制作する。
案件ごとに `{クライアント名}/` フォルダを作成し、その中で作業する。

## フォルダ・ファイル規則

- クライアントフォルダ: `{クライアント名}/`（正式名称、スペースなし）
- 標準構成: `index.html` / `css/style.css` / `js/main.js` / `images/`
- ファイル名はすべて小文字・ハイフン区切り（例: `about-us.html`）
- 新規ページは `{クライアント名}/` 直下に配置

## 技術スタック（デフォルト）

- 素のHTML5 / CSS3 / JavaScript（バニラ）
- 外部ライブラリは最小限。使う場合はCDNを優先
- モバイルファーストのレスポンシブデザイン
- 画像は `images/` フォルダに格納

## コーディング規則

- インデント: スペース2つ
- CSSはセクションごとにコメントで区切る
- JavaScriptは `'use strict'` を先頭に記載
- HTMLのlang属性は `ja`

## エージェント連携

複雑な案件では以下の順序でエージェントを活用する:
1. マネージャー → 要件整理・構成計画
2. デザイナー → HTML/CSS制作
3. エンジニア → JS・機能実装
4. ライター → コンテンツ作成
5. テスター・批評家 → 品質レビュー

## 禁止事項

- クライアント情報を他フォルダに漏らさない
- 未確認の外部APIやサービスを無断で組み込まない
- 既存ファイルを確認せずに上書きしない
